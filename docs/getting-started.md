# Getting Started

If you're ready to get started with Security Onion, you may have questions like:

**What are the recommended best practices?**

See the [Best Practices](best-practices.md) section.

**How many machines do I need?**

Depending on what you're trying to do, you may need anywhere from one machine to thousands of machines. The [Use Cases](use-cases.md) and [Architecture](architecture.md) sections will help you decide.

**What kind of hardware does each of those machines need?**

This could be anything from a small virtual machine to a large rack mount server with lots of CPU cores, lots of RAM, and lots of storage. The [Hardware](hardware.md) section provides further details.

**If I just want to try Security Onion in a virtual machine, how do I create a virtual machine?**

See the [VMware](vmware.md), [VirtualBox](virtualbox.md), and [Proxmox](proxmox.md) sections.

**How do I deploy Security Onion in the cloud?**

See the [Amazon Cloud](cloud-amazon.md), [Azure Cloud](cloud-azure.md), and [Google Cloud](cloud-google.md) sections.

**What if I have trouble booting the ISO image?**

Check out the [Trouble Booting](trouble-booting.md) section.

**What if I'm on an airgap network?**

Review the [Airgap](airgap.md) section.

**Once I've booted the ISO image, how do I install it?**

See the [Installation](installation.md) section.

**After installation, how do I configure Security Onion?**

The [Configuration](configuration.md) section covers many different use cases.

**Is there anything I need to do after configuration?**

See the [Post Installation](post-installation.md) section.
